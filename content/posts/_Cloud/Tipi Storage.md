### Volume Storage
disco rigido tradizionale (bassa latenza , dati strutturati)

### Block storage 
tipo disco per nostra vm (volume asseccesibile solo da SO)

### File storage
struttura gerarchica : ottima performance  e per codivisione file non scalabili

### Object Storage
oggetto = file + metadati  facile scabilita e dati non strutturati  , accessibile da API , oggetti non modificabili
